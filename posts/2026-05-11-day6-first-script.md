# 2026-05-11 — day6-first-script

## LinkedIn

I run a Sales Development Team.

I also write scripts.

Today one of them told me to go f**k myself.

zsh: permission denied: ./newpost

Eeek!

A year ago, that would have ended the day

Today, it took me 90 seconds to fix

The script I built is a tool that outlines my daily LinkedIn post:

- date stamped
- sections prefilled

Ready to draft in 10 seconds instead of 10 minutes

And the reason I fixed that error so fast is because I learned the thing most sales leaders will never learn:

The error is the instruction

"Permission denied" wasn't telling me I broke something

It was telling me macOS hadn't been given the proper permissions to run the file

I executed a quick command (chmod +x) and the script worked

Most people outsource everything technical because the first error feels like they hit a wall (or exploded their computer)

It's not

It's just a sentence

Read it, run the fix, move on

The reps I'm hiring now aren't the ones who know the most plays

They're the ones who don't quit when the screen turns red

What's the last "permission denied" moment that stopped you?
